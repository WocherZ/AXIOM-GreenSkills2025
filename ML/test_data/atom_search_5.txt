1. Общее описание
Атом.Око — это импортонезависимое программное решение для распознавания, анализа и обработки документов, разработанное компанией Гринатом (входит в структуру Росатома). Оно применяет технологии машинного обучения для точного извлечения информации из документов, проверки данных и автоматизации документооборота в различных сферах бизнеса 【10】.

2. Функциональные возможности
Распознавание документов: Обрабатывает сканированные документы и фотографии, извлекает текст даже при плохом освещении или сложных углах съемки. Ведутся работы по внедрению распознавания рукописного текста 【8】.
Интеллектуальный анализ: Позволяет проводить перекрестные и арифметические проверки внутри комплекта документов 【10】.
Автоматизация документооборота: Взаимодействует с RPA-платформой Атом.РИТА, что позволяет автоматически извлекать данные, сверять их с бухгалтерскими записями и формировать отчеты 【9】.
Гибкость: Позволяет пользователю самостоятельно добавлять новые типы документов и настраивать логику обработки 【10】.
Вопросно-ответная система: Функция «Поговори с документом» позволяет пользователю загружать файл и запрашивать у системы нужную информацию без необходимости читать весь документ 【8】.
3. Производительность
Система обладает высокой точностью распознавания, которая повышается по мере использования и адаптации к различным типам документов. Однако, как и в любом OCR-решении, возможны ошибки, которые требуют верификации человеком 【8】.

Интеграция с другими системами осуществляется через гибкий API, что делает переход на платформу удобным и минимизирует затраты времени и ресурсов 【8】.

4. Применение

Техническое описание проекта: Система распознавания документов и текста с использованием OCR (включая рукописный текст)
1. Введение
Проект представляет собой систему автоматического распознавания текста с документов, включая печатный и рукописный текст, с использованием технологий оптического распознавания символов (OCR). Система предназначена для обработки сканированных документов, фотографий и других изображений с текстом, с целью извлечения и структурирования информации для дальнейшего анализа или интеграции в другие системы.

2. Цели и задачи
Цель: Создание высокоточного решения для распознавания текста (печатного и рукописного) с различных типов документов.

Задачи:

Предобработка изображений для улучшения качества распознавания.

Распознавание печатного текста с высокой точностью.

Распознавание рукописного текста с использованием современных моделей машинного обучения.

Постобработка текста для исправления ошибок и структурирования данных.

Интеграция системы в существующие workflows (например, системы документооборота).

3. Архитектура системы
Система состоит из следующих модулей:

Модуль предобработки изображений:

Нормализация изображений (устранение шумов, коррекция освещения, поворотов и искажений).

Бинаризация (перевод изображения в черно-белый формат).

Сегментация текстовых блоков, строк и символов.

Модуль распознавания печатного текста:

Использование OCR-движков для извлечения текста.

Поддержка многоязычности.

Модуль распознавания рукописного текста:

Использование нейронных сетей для обработки рукописного ввода.

Поддержка различных стилей почерка.

Модуль постобработки текста:

Исправление ошибок распознавания.

Структурирование данных (например, извлечение ключевых полей: ФИО, дата, номер документа и т.д.).

Интерфейс взаимодействия:

REST API для интеграции с другими системами.

Веб-интерфейс для загрузки и просмотра результатов.

4. Используемые технологии и модели
Предобработка изображений:

OpenCV: для коррекции изображений (повороты, бинаризация, фильтрация шумов).

Tesseract: для сегментации текстовых блоков.

Распознавание печатного текста:

Tesseract OCR: основной OCR-движок с поддержкой множества языков.

EasyOCR: альтернативный OCR-движок с высокой точностью для печатного текста.

Распознавание рукописного текста:

CRNN (Convolutional Recurrent Neural Network): модель для распознавания последовательностей (например, строк рукописного текста).

Transformer-based модели: например, модель TrOCR (Transformer-based OCR), которая показывает высокие результаты для рукописного текста.

HWR (Handwriting Recognition) модели: например, IAM Handwriting Database для обучения моделей на датасетах рукописного текста.

Постобработка текста:

Языковые модели: например, BERT или GPT для исправления ошибок и контекстного анализа текста.

Регулярные выражения: для извлечения структурированных данных (например, дат, номеров телефонов).

Инфраструктура:

Python: основной язык разработки.

5. Этапы работы системы

Печатный текст распознается с помощью Tesseract/EasyOCR.

Рукописный текст обрабатывается с использованием CRNN или TrOCR.

Постобработка:

Текст исправляется с помощью языковых моделей.

Извлекаются структурированные данные.

Вывод результата:

Результат возвращается в формате JSON (для API) или отображается в веб-интерфейсе.
